# Python 虛擬環境 (venv) 與 VS Code 使用教學

## 什麼是虛擬環境？

虛擬環境是Python專案的獨立環境，可以隔離不同專案的依賴套件，避免版本衝突。類似於其他語言的包管理器。

## 基本使用步驟

### 1. 創建虛擬環境

在專案根目錄下執行：
```bash
python -m venv projvenv
```
這會創建一個名為 `projvenv` 的資料夾，包含Python解釋器和pip。

### 2. 啟動虛擬環境

#### Windows:
```bash
.\projvenv\Scripts\activate
```

#### macOS/Linux:
```bash
source projvenv/bin/activate
```

啟動後，命令提示符會顯示 `(projvenv)` 前綴。

### 3. 停用虛擬環境

```bash
deactivate
```

### 4. 安裝套件

啟動環境後，使用pip安裝套件：
```bash
pip install package_name
```

## VS Code 中的使用

### 設定Python解釋器

1. 開啟VS Code，進入專案資料夾
2. 按 `Ctrl+Shift+P` (Windows) 或 `Cmd+Shift+P` (Mac) 開啟命令面板
3. 輸入 "Python: Select Interpreter"
4. 選擇虛擬環境中的Python解釋器（通常在 `myenv/Scripts/python.exe` 或 `myenv/bin/python`）

### 確認切換成功方式

在終端機中執行 `python --version` 應該會顯示虛擬環境中的Python版本。

### 自動啟動環境

VS Code 會自動在終端機中啟動選定的虛擬環境，你可以在終端機中直接執行Python程式。

### 設定檔

在專案根目錄創建 `.vscode/settings.json`：
```json
{
    "python.defaultInterpreterPath": "./myenv/Scripts/python.exe"
}
```

## 常用命令

- 查看已安裝套件：`pip list`
- 匯出依賴：`pip freeze > requirements.txt`
- 從檔案安裝：`pip install -r requirements.txt`
- 刪除環境：直接刪除 `myenv` 資料夾

## 注意事項

- 不要將虛擬環境資料夾提交到Git（加入 `.gitignore`）
- 每個專案使用獨立的虛擬環境
